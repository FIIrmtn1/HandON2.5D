# วิธีทำเกมทั้งเกม (ภาพรวม + อ้างอิงโปรเจกต์ HandON2.5D)

Checklist ภาพรวมขั้นตอนทำเกมตั้งแต่ต้นจนจบ แต่ละข้อมีโน้ตสั้นๆ ว่าในโปรเจกต์นี้ (HandON2.5D, Unity, 2.5D) อยู่ตรงไหน/ควรทำอะไรต่อ

## 1. ไอเดียและแนวทาง (Concept)
- กำหนดแนวเกม, เป้าหมายผู้เล่น, จุดขาย 1 ประโยค
- โปรเจกต์นี้: เกม 2.5D แนวหลบหนี/ซ่อนตัว มีตัวละครสไปรต์ 2D เดินอยู่บนฉาก 3D (ระบบ "quiet/noise", hook grapple, snap dash)

## 2. ออกแบบเกม (Game Design)
- กำหนดกลไกหลัก (core loop), ด่าน/บท, ศัตรู, เงื่อนไขแพ้-ชนะ
- โปรเจกต์นี้: มีบทแบบห้อง/ฉาก (`Ch1_Bedroom`, `Ch3_Kitchen`) แต่ละฉากมี zone, จุดเสี่ยง (ศัตรู Pudding), จุดหนี (ExitTrigger)

## 3. วางโครงสร้างโปรเจกต์ (Project Setup)
- ตั้งโฟลเดอร์ให้เป็นระบบ: Scenes, Scripts, Sprites, Materials, Prefabs
- แยกสคริปต์ที่ทำหน้าที่ต่างกันให้ชัด (เช่นในโปรเจกต์นี้แยกชุด 2D กับ 3D ของ PlayerController ไว้คนละไฟล์)

## 4. เขียนระบบแกนหลัก (Core Systems / Programming)
- ระบบเคลื่อนที่ตัวละคร, กล้อง, การชนกัน (physics/collider)
- ระบบเฉพาะเกม เช่น noise/detection, grapple hook, dash
- โปรเจกต์นี้: `PlayerController3D`, `HookGrapple3D`, `SnapDash3D`, `QuietHang3D`, `CameraFollow3D`

## 5. ทำ Art และ Animation
- วาด/หาภาพตัวละคร, ทำ sprite sheet, ตัด frame
- ทำ Animator Controller + Animation Clip ผูกกับสถานะตัวละคร (เดิน/หยุด)
- โปรเจกต์นี้: `bendysheet.png` → ตัด 6 เฟรม → `BEndyanim.anim` (เดิน) + `BEndyIdle.anim` (หยุดนิ่ง) → ผูกผ่าน `bendysheet_0.controller` ด้วย parameter `IsWalking`

## 6. ประกอบฉาก (Level/Scene Building)
- จัดวาง GameObject จริงในฉาก: พื้น, สิ่งกีดขวาง, จุดเกิด, แสง, กล้อง
- แยก physics root กับ visual (sprite/model) ออกจากกัน เพื่อไม่ให้ scale ภาพไปกระทบ collider

## 7. เสียง (Audio)
- เพลงประกอบ, เสียงเอฟเฟกต์ (ฝีเท้า, จับได้, เปิดประตู) — ยังไม่มีในโปรเจกต์นี้ ต้องทำเพิ่มถ้าต้องการ

## 8. UI/UX
- เมนูหลัก, HUD, prompt กดปุ่ม (เช่น `HookPromptUI3D`, `ChargeIndicator3D` ที่มีอยู่แล้ว), หน้าจอแพ้-ชนะ

## 9. เชื่อมทุกอย่างเข้าด้วยกันแล้วทดสอบ (Integration + Playtest)
- เล่นจริงใน Play Mode ทุกครั้งหลังแก้ระบบสำคัญ
- เช็คจุดต่อรอยระหว่างระบบ (เช่น แอนิเมชันต้องซิงก์กับการเคลื่อนที่จริง ไม่ใช่วนลูปเอง)
- ห้าม save scene ระหว่างอยู่ใน Play Mode และการแก้ไขระหว่าง Play Mode จะหายเมื่อออกจากโหมด

## 10. ปรับสมดุลและแก้บั๊ก (Balancing + Bug Fixing)
- ปรับความเร็ว/ความยาก/ตำแหน่งจุดเกิดศัตรู จากผลการเทสต์จริง
- แก้บั๊กที่ต้นเหตุ (เช่น ตัวละครลอยเพราะ scale ผิดที่ ไม่ใช่ตำแหน่งผิด)

## 11. บันทึกและจัดการเวอร์ชัน (Save + Version Control)
- save scene/asset เป็นระยะ และ commit ผ่าน git เพื่อย้อนดูความคืบหน้าได้
- สำรองไฟล์ต้นฉบับก่อนแก้ไขแบบทำลายล้าง เช่น การแก้ภาพ sprite ตรงๆ

## 12. Build และเผยแพร่ (Build + Release)
- ตั้งค่า Player Settings, Build Settings, ทดสอบ build จริงบนเครื่องเป้าหมาย
- ยังไม่ถึงขั้นนี้ในโปรเจกต์นี้ ณ ตอนนี้ — ทำหลังจากเนื้อหาเกมเล่นจบได้ครบอย่างน้อย 1 บท

## 13. ทำซ้ำ (Iterate)
- เก็บ feedback แล้ววนกลับไปข้อ 2-10 ตามความจำเป็น การทำเกมไม่ใช่เส้นตรง มักต้องย้อนแก้ดีไซน์หลังเห็นผลจริงจากการเทสต์
